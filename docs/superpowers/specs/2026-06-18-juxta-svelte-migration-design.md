# Juxta — Design Spec

**Date:** 2026-06-18
**Status:** Approved
**Topic:** Migrate DiffStudio (React/npm/Vite) to Juxta (SvelteKit/Bun) as a self-hostable, open-source alternative to text-compare.com.

## 1. Goal & context

The current app ("DiffStudio" / metadata title "Text Compare") is a 100% client-side React 19 + Vite + Tailwind v4 diff utility generated by Google AI Studio. All logic lives in two files: a ~2600-line monolith [src/App.tsx](../../../src/App.tsx) and a pure diff engine [src/utils/diff.ts](../../../src/utils/diff.ts).

This project rewrites it as **Juxta**: a self-hostable, AGPL-3.0, open-source side-by-side text/code diff tool — an open alternative to text-compare.com — built on SvelteKit + Bun, with full feature parity, a real test suite, accessibility, responsiveness, Docker self-hosting, and complete OSS documentation.

## 2. Locked decisions

| Decision              | Choice                                          | Rationale                                                                         |
| --------------------- | ----------------------------------------------- | --------------------------------------------------------------------------------- |
| Name                  | **Juxta**                                       | From "juxtapose" — place side by side; short, brandable, on-theme                 |
| Framework             | **SvelteKit + Svelte 5 (runes)**                | User-directed migration target                                                    |
| Runtime / pkg manager | **Bun**                                         | User-directed; native test runner                                                 |
| SvelteKit adapter     | **adapter-node**                                | Self-hostable server; room for a future server-side diff API                      |
| Feature scope         | **Full 1:1 parity**                             | No functionality regression vs current app                                        |
| Styling               | **Keep Tailwind v4**                            | Minimal churn; preserves existing visual design                                   |
| Component lib         | **shadcn-svelte / bits-ui (selective)**         | Accessible headless primitives for chrome; copy-in (owned code); custom diff core |
| Animation             | **Svelte transitions** (replace `motion/react`) | Native, `prefers-reduced-motion`-aware                                            |
| Unit tests            | **bun test**                                    | Fastest, zero extra deps, pairs with Bun                                          |
| E2E / a11y tests      | **Playwright + axe-core**                       | Mature browser + accessibility coverage                                           |
| License               | **AGPL-3.0**                                    | Network-use copyleft protects the self-hostable open-alternative goal             |
| CI                    | **GitHub Actions**                              | Standard, free for public repos                                                   |
| Diff location (v1)    | **Client-side**                                 | Unchanged from today; server endpoint deferred                                    |

## 3. Architecture

SvelteKit application served by `adapter-node`, run with Bun. Diffing remains entirely client-side for v1; the Node server is the static/SSR host and a seam for a future `/api/diff` endpoint. Svelte 5 runes (`$state` / `$derived` / `$effect`) replace React hooks. The pure diff engine ports over with **zero logic changes**.

### Project structure

```text
src/
  routes/
    +layout.svelte          # theme class wiring, global chrome
    +page.svelte            # composes the app; hosts use:shortcuts action
  lib/
    diff/
      engine.ts             # diffLines / diffTokens / alignDiff (port of utils/diff.ts, unchanged logic)
      tokenizer.ts          # tokenizeLanguage (ts/html/css/json/plain)
      detect.ts             # detectLanguage
    export/
      html.ts | txt.ts | md.ts | json.ts   # report generators (one per format)
      index.ts              # exportReport(format) dispatcher + download trigger
    samples.ts              # demo samples (code/prose/html/css/json)
    state/
      settings.svelte.ts    # createPersisted rune helper + all persisted prefs
    components/
      Toolbar.svelte
      EditorPane.svelte         # input textarea + per-side transform menu + copy
      DiffView.svelte           # split/unified container, scroll sync
      DiffRow.svelte            # one aligned row
      SyntaxLine.svelte         # tokenized + search-highlighted line rendering
      SearchBar.svelte
      ExportMenu.svelte         # bits-ui dropdown
      TextTransformMenu.svelte  # bits-ui dropdown
      LanguageSelect.svelte     # bits-ui select
      ThemeToggle.svelte
      StatsBar.svelte           # additions/deletions/modifications/similarity/parse time
      SampleLoader.svelte
      ShortcutsModal.svelte     # bits-ui dialog (focus trap, Escape)
  lib/ui/                       # shadcn-svelte generated primitives (dialog, dropdown-menu, select, toggle, tooltip, button)
  app.css                       # Tailwind entry
```

The monolith becomes ~12 focused components, each independently testable.

## 4. Component & responsibility breakdown

- **diff/engine.ts** — verbatim port of `diffLines`, `diffTokens`, `tokenize`, `alignDiff`, including identical-prefix/suffix stripping and the `nMid * mMid < 12_250_000` safety cap that falls back to naive delete-then-insert. Types: `DiffOp`, `AlignedDiffRow`, `DiffWord`.
- **diff/tokenizer.ts** — `tokenizeLanguage(text, lang)` regex tokenizer for `ts | html | css | json | plain`; pure, no DOM.
- **diff/detect.ts** — `detectLanguage(text)` heuristics.
- **export/** — the four report builders split out of `exportReport`; each takes the aligned rows + stats and returns a string; `index.ts` handles MIME + download.
- **state/settings.svelte.ts** — `createPersisted<T>(key, default)` rune wrapping `localStorage` (replaces ~12 repetitive React effects). Keys rebrand `text-compare-*` → `juxta-*`. Holds: theme, compareMode, syncScroll, showWhitespace, showLineNumbers, lineWrap, highlightMode, syntaxHighlighting, syntaxScheme, responsiveLayout, autoCompare, languageMode.
- **components/** — presentational + interaction; the diff result is a `$derived` of committed-left/right text + highlight mode, preserving today's auto-compare vs manual-commit ("dirty") model. Keyboard shortcuts live in a `use:shortcuts` Svelte action mirroring the current Alt-key map (Alt+N/P jump, W wrap, L line numbers, M word/char, K shortcuts, E export, C clear, Enter compare, X swap, F search-focus).

## 5. Feature parity checklist

Split & unified views · scroll sync · 5-language syntax highlighting (ts/html/css/json/plain) + auto-detect · word/char inline diff · search with next/prev navigation · whitespace + line-wrap + line-number toggles · soft/high-contrast syntax schemes · per-side text transforms (lowercase, sort lines, remove excess whitespace, replace line breaks) · swap sides · clear · 4-format export (HTML/TXT/MD/JSON) · localStorage-persisted prefs · keyboard shortcuts + shortcuts modal · sample loader · light/dark theme · stats (additions/deletions/modifications/similarity %/parse time) · adaptive vs fixed-split responsive layout.

## 6. Responsiveness & accessibility (target: WCAG 2.1 AA)

- Split view collapses to stacked/unified below a breakpoint; toolbar wraps gracefully; existing `adaptive` vs `fixed-split` pref drives behavior.
- Semantic landmarks; ARIA labels on every icon-only button; focus-trap + Escape on the shortcuts modal (via bits-ui dialog); visible focus rings; full keyboard operability of all chrome.
- Audit diff add/delete/modify colors for contrast in **both** themes (today's custom shades — e.g. `slate-550`, `amber-750` — get verified/adjusted).
- Respect `prefers-reduced-motion`. Axe scans in Playwright gate a11y regressions.

## 7. Testing

- **bun test (unit):** diff engine (diffLines/diffTokens/alignDiff incl. prefix-suffix + safety-cap fallback), tokenizer, language detection, text transforms, export generators. Deterministic pure logic — highest value.
- **Playwright (E2E + a11y):** paste→diff, split/unified toggle, search navigation, export download, theme persistence across reload, keyboard shortcuts, axe scans on key UI states. (Svelte component tests kept light per the bun-test caveat; UI behavior covered by Playwright.)

## 8. Docker & self-hosting

Multi-stage Dockerfile: `oven/bun` build stage (`bun install`, `bun run build`) → slim runtime running the adapter-node output (`bun ./build`). `docker-compose.yml`, `.dockerignore`, healthcheck, configurable `PORT`/`HOST`/`ORIGIN`. README self-host quickstart: single `docker run` line + compose snippet.

## 9. OSS documentation & CI

- `README.md` — rewritten: positioning ("open, self-hostable alternative to text-compare.com"), feature list, screenshot, self-host quickstart (Docker + compose), local dev (Bun), tech stack.
- `LICENSE` — AGPL-3.0 full text.

## 10. Migration sequencing

1. **Skeleton** — scaffold SvelteKit + Bun + Tailwind v4 + adapter-node + shadcn-svelte init; `git init` (done); base config.
2. **Engine + unit tests** — port `diff/` and `export/`; write the `bun test` suite. Pure, parallelizable, no UI dependency.
3. **Components** — port the UI to full parity, wiring runes state + bits-ui chrome.
4. **Parallel (subagent-driven-development)** — once the app runs: (a) responsiveness + a11y pass, (b) Playwright E2E + axe, (c) Docker, (d) OSS docs, (e) GitHub Actions CI. Largely independent.
5. **Cutover** — remove React/Vite/old `src/App.tsx`, `src/main.tsx`, `index.html` React entry, `vite.config.ts`, npm `package.json` deps; finalize README; update root `CLAUDE.md` to describe the Svelte stack.

## 11. Out of scope (v1)

Server-side diff API · file upload / drag-drop file compare · diff-by-URL / shareable links · auth / multi-user · PDF export · real-time collaboration. (The adapter-node choice leaves room for the server API later.)

## 12. Risks & mitigations

- **shadcn-svelte theme vs hand-tuned palette** — adopt primitives only; restyle to existing look; don't import default theme wholesale.
- **bun test + @testing-library/svelte maturity** — keep component tests light; cover UI behavior via Playwright.
- **Tailwind v4 + shadcn-svelte compatibility** — verify versions at skeleton step before building components on them.
- **Feature-parity drift** — the §5 checklist is the acceptance gate; the migration isn't complete until every item is verified in the Svelte app.
